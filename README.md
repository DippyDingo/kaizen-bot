# KAIZEN Bot

Telegram-бот для личной продуктивности в формате single-message dashboard с RPG-метриками.

README отражает фактическое состояние репозитория на **6 марта 2026** и сверён с [PLAN.md](PLAN.md). Ниже указано, что уже есть в коде, что реализовано частично и что пока остаётся в плане.

## Текущее состояние

Сейчас проект состоит из двух рабочих слоёв:
- `bot/` — интерфейс в Telegram на `aiogram 3`
- `backend/` — модели, async-доступ к БД и бизнес-сервисы на `SQLAlchemy`

Текущий MVP работает вокруг одного обновляемого сообщения в личном чате Telegram и закрывает базовые сценарии:
- задачи,
- календарь,
- дневник,
- вода,
- all-time статистика,
- профиль и имя пользователя.

## Что уже реализовано

### База данных и миграции
- Настроен async-слой БД через `SQLAlchemy`.
- Настроен `Alembic`.
- Есть миграции:
  - `20260306_000001_init_mvp`
  - `20260306_000002_add_diary_entries`
  - `20260306_000003_expand_diary_media`
  - `20260306_000004_add_user_preferred_name`
- В БД уже используются модели:
  - `User`
  - `Task`
  - `WaterLog`
  - `SleepLog`
  - `Habit`
  - `DiaryEntry`

### Telegram UI
- Основной сценарий реализован как `single-message UI`: бот обновляет один dashboard вместо спама новыми сообщениями.
- Интерфейс ограничен приватными чатами, чтобы логика dashboard не ломала группы.
- Глобальная навигация вынесена в быстрые кнопки у поля ввода:
  - `🏠 Главная`
  - `📋 Задачи`
  - `📝 Дневник`
  - `📅 Календарь`
  - `📊 Статистика`
  - `❤️ Здоровье`
  - `⚙️ Настройки`
- Нижняя системная кнопка Telegram используется под `Web App` через `WEBAPP_URL`.
- Slash-команды у бота очищаются и не используются как основной UI.

### Главная
- Показывает дату, отображаемое имя пользователя, уровень, EXP, streak.
- Есть три прогресс-зоны:
  - HP по задачам/привычкам
  - Мана по воде
  - Стамина по сну
- После добавления или отката воды главная перерисовывается сразу, мана обновляется интерактивно.
- Есть быстрые действия:
  - `💧150`
  - `💧250`
  - `💧500`
  - `↩️ Вода`
  - `➕ Задача`
  - `➕ Запись`

### Задачи
- Добавление задачи через FSM:
  - название
  - приоритет
  - дата через календарь
- Приоритеты:
  - `high`
  - `medium`
  - `low`
- Список задач сортируется так:
  - сначала невыполненные,
  - затем `high -> medium -> low`,
  - затем по порядку создания.
- Кнопка самой задачи переключает статус выполнения.
- Удаление находится справа от задачи отдельной кнопкой `❌`.
- Слева у задачи показывается маркер приоритета.
- Полные длинные названия задач дублируются в текстовом блоке экрана, чтобы не терялись из-за ограничений ширины inline-кнопок.

### Календарь
- Реализован выбор дня и навигация по месяцам.
- Календарь используется в двух сценариях:
  - просмотр задач по дате,
  - выбор даты при создании задачи.
- Есть отдельный календарный режим для дневника.

### Дневник
- Поддерживаются записи типов:
  - текст,
  - голосовое,
  - кружок,
  - фото,
  - видео.
- Можно:
  - добавить запись,
  - открыть конкретную запись,
  - выгрузить все записи за выбранный день в чат,
  - очистить отправленные ботом сообщения выгрузки.
- Список записей и счётчики за день уже работают.

### Здоровье
- Сейчас из health-блока реально реализована вода.
- Есть быстрое добавление воды по фиксированным объёмам.
- Есть undo последнего добавления воды за выбранный день.
- Сон участвует в расчёте стамины и статистики, но полноценного UI для сна пока нет.

### Статистика
- Экран статистики показывает агрегаты **за всё время**, а не за выбранный день.
- Сейчас выводятся:
  - дата начала использования,
  - уровень,
  - текущий EXP,
  - текущий и максимальный streak,
  - всего задач / выполнено задач / процент,
  - всего воды,
  - всего сна,
  - всего записей дневника.

### Настройки и профиль
- Добавлен отдельный раздел `⚙️ Настройки`.
- Профиль перенесён туда.
- В профиле есть:
  - текущее отображаемое имя,
  - Telegram-имя,
  - username,
  - дата начала использования,
  - смена имени.
- При первом `/start`, если имя ещё не задано, бот спрашивает: как к тебе обращаться.
- Используется `preferred_name`, если оно сохранено.

### Структура обработчиков
- Старый монолитный обработчик разнесён по тематическим модулям:
  - `bot/handlers/single_message_parts/common.py`
  - `bot/handlers/single_message_parts/core.py`
  - `bot/handlers/single_message_parts/tasks.py`
  - `bot/handlers/single_message_parts/calendar.py`
  - `bot/handlers/single_message_parts/diary.py`
  - `bot/handlers/single_message_parts/health.py`
- `bot/handlers/single_message.py` оставлен как совместимый wrapper для подключения роутера.

## Что реализовано частично

Эти блоки уже отражены в коде или интерфейсе, но ещё не закрыты полностью:
- `Web App`: нижняя кнопка Telegram уже настроена, но полноценного фронтенда и backend API в репозитории нет.
- `RPG`: уровень, EXP и streak работают, но полноценные игровые системы из плана пока не построены.
- `Здоровье`: вода есть, сон только как метрика, остальной health-модуль из плана не реализован.
- `Статистика`: есть базовые all-time агрегаты, но нет графиков, сравнений, weekly/monthly analytics.

## Что логично делать дальше

Ближайшие практические шаги по текущему состоянию проекта:
1. Довести `Здоровье` до полноценного раздела: сон, тренировки, дополнительные метрики и UI.

## Короткий roadmap

### MVP
- стабилизировать текущий Telegram UI;
- закрыть базовые сценарии по задачам, дневнику, воде, профилю и all-time статистике;
- довести раздел `Здоровье` до минимально целостного состояния.

- добавить полноценные сценарии сна и тренировок.

## Быстрый запуск

### 1. Подготовить переменные окружения
Создай `.env` на основе `.env.example` и заполни минимум:
- `BOT_TOKEN`
- `DATABASE_URL`
- `WEBAPP_URL`

### 2. Поднять PostgreSQL
```bash
docker compose up -d postgres
```

### 3. Применить миграции
```bash
alembic upgrade head
```

### 4. Запустить бота
```bash
python -m bot.main
```

Для Windows с локальным venv:
```powershell
.\.venv\Scripts\python.exe -m bot.main
```

## Полезные команды
