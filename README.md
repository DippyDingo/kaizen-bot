# KAIZEN Bot

Telegram-бот для личной продуктивности в формате single-message dashboard с RPG-метриками.

README отражает фактическое состояние репозитория на **7 марта 2026** и сверяется с `PLAN.md`: ниже отдельно перечислено, что уже есть в коде, что реализовано частично и что пока остается только в плане.

## Текущее состояние

Проект сейчас состоит из двух рабочих слоев:
- `bot/` — интерфейс в Telegram на `aiogram 3`
- `backend/` — модели, async-доступ к БД и бизнес-сервисы на `SQLAlchemy`

Текущий MVP работает вокруг одного обновляемого сообщения в личном чате Telegram и уже закрывает базовые сценарии:
- главная панель,
- задачи,
- календарь,
- дневник,
- здоровье,
- статистика,
- профиль и настройки.

## Что уже реализовано

### База данных и миграции
- Настроен async-слой БД через `SQLAlchemy`.
- Настроен `Alembic`.
- Есть миграции:
  - `20260306_000001_init_mvp`
  - `20260306_000002_add_diary_entries`
  - `20260306_000003_expand_diary_media`
  - `20260306_000004_add_user_preferred_name`
  - `20260306_000005_add_workout_logs`
  - `20260306_000006_add_user_daily_targets`
  - `20260306_000007_add_medication_logs`
  - `20260306_000008_add_medication_courses_and_schedule_status`
  - `20260306_000009_add_dashboard_message_ref`
  - `20260306_000010_add_wellbeing_logs`
- В БД используются модели:
  - `User`
  - `Task`
  - `WaterLog`
  - `SleepLog`
  - `WorkoutLog`
  - `MedicationCourse`
  - `MedicationLog`
  - `WellbeingLog`
  - `Habit`
  - `DiaryEntry`

### Telegram UI
- Основной сценарий реализован как `single-message UI`: бот обновляет один dashboard вместо спама новыми сообщениями.
- Интерфейс ограничен приватными чатами, чтобы логика dashboard не ломала группы.
- Ссылка на активный dashboard сохраняется и в FSM state, и в БД, поэтому после повторного входа и после рестарта бот может восстановить целевое сообщение для редактирования.
- Глобальная навигация вынесена в быстрые кнопки у поля ввода:
  - `🏠 Главная`
  - `📋 Задачи`
  - `📝 Дневник`
  - `📅 Календарь`
  - `📊 Статистика`
  - `❤️ Здоровье`
  - `⚙️ Настройки`
- Нижняя системная кнопка Telegram используется под `Web App` через `WEBAPP_URL`.
- Slash-команды очищаются и не используются как основной UI.
- Reply keyboard держится на отдельном carrier-сообщении. После удаления истории чат-клиент Telegram убирает и его, поэтому корректное восстановление делается через новый `/start`.
- На `/start` бот принудительно сбрасывает кеш UI чата, перевешивает carrier-кнопки и переносит dashboard к текущей точке диалога.

### Главная
- Показывает дату, имя пользователя, уровень, EXP, streak и короткий companion-блок.
- Есть три статусные зоны:
  - `❤️ Ритм`
  - `💧 Вода`
  - `⚡️ Сон`
- После добавления или отката воды главный экран перерисовывается сразу.
- Есть быстрые действия:
  - `💧150`
  - `💧250`
  - `💧500`
  - `↩️ Вода`
  - `➕ Задача`
  - `➕ Запись`

### Задачи
- Добавление задачи через FSM:
  - название,
  - приоритет,
  - дата через календарь.
- Приоритеты:
  - `high`
  - `medium`
  - `low`
- Список задач сортируется так:
  - сначала невыполненные,
  - затем `high -> medium -> low`,
  - затем по времени создания.
- Нажатие на кнопку задачи переключает статус выполнения.
- Удаление вынесено отдельной кнопкой справа.
- Слева у задачи показывается маркер приоритета.
- Длинные названия дополнительно выводятся в текстовом блоке экрана.

### Календарь
- Реализован выбор дня и навигация по месяцам.
- Календарь используется в двух сценариях:
  - просмотр задач по дате,
  - выбор даты при создании задачи.
- Есть отдельный календарный режим для дневника.

### Дневник
- Поддерживаются записи типов:
  - текст,
  - голосовое,
  - кружок,
  - фото,
  - видео.
- Можно:
  - добавить запись,
  - открыть конкретную запись,
  - выгрузить все записи за выбранный день в чат,
  - очистить сообщения выгрузки, отправленные ботом.
- Работают дневные списки и счетчики записей.

### Здоровье
- Раздел разбит на два summary-экрана:
  - `День`
  - `Неделя`
- Реализована вода:
  - быстрые кнопки добавления,
  - откат последнего добавления за день,
  - дневные и недельные сводки,
  - прогресс-бары,
  - пользовательский дневной лимит воды из настроек.
- Реализован сон:
  - запись по длительности,
  - запись по точному времени,
  - оценка качества сна,
  - откат последней записи,
  - дневные и недельные сводки,
  - прогресс-бары.
- Начат и уже работает блок тренировок:
  - выбор типа тренировки,
  - выбор длительности,
  - свободный ввод длительности (`25` или `1:15`),
  - запись тренировки кнопками,
  - откат последней тренировки за день,
  - дневные и недельные сводки по тренировкам,
  - пользовательский дневной лимит тренировок из настроек.
- Реализован блок лекарств:
  - отдельное medication-окно внутри `Здоровья`,
  - создание курса `название -> доза -> время -> сколько дней`,
  - старт курса от выбранной даты,
  - календарь лекарств с отметками дней,
  - отметка `выпил / пропуск` на конкретный день,
  - отключение курса,
  - дневная и недельная сводка по лекарствам.
- Реализован блок состояния:
  - кнопочный flow `энергия -> стресс`,
  - одна запись состояния на день с обновлением вместо дублей,
  - дневная оценка `энергии / стресса`,
  - недельные средние по `энергии / стрессу`,
  - встроено в `Здоровье` без отдельного top-level раздела.

### Статистика
- Поддерживаются периоды:
  - `День`
  - `7 дней`
  - `30 дней`
  - `Всё время`
- Экран разбит на читаемые блоки:
  - `Общее`
  - `Задачи`
  - `Вода`
  - `Сон`
  - `Дневник`
- Есть визуальные бары для метрик с понятной шкалой.
- Сейчас считаются:
  - задачи: всего, выполнено, процент, активные дни, разрез по приоритетам,
  - вода: сумма, среднее, активные дни, лучший день,
  - сон: сумма, среднее, качество, активные дни, лучший день, самый длинный сон,
  - тренировки: сумма, среднее, активные дни, лучший день, разрез по типам и минутам,
  - лекарства: приемы, выпито, пропуски, активные дни, уникальные лекарства, лучший день, самое частое лекарство,
  - состояние: записи, активные дни, средняя энергия, средний стресс, лучший день по энергии, самый стрессовый день,
  - дневник: общее число записей, активные дни, лучший день.

### Настройки и профиль
- Есть отдельный раздел `⚙️ Настройки`.
- Профиль вынесен туда.
- В настройках можно менять:
  - отображаемое имя,
  - дневной лимит воды,
  - дневной лимит тренировок.
- В профиле доступны:
  - отображаемое имя,
  - Telegram-имя,
  - username,
  - дата начала использования,
  - смена имени.
- При первом `/start`, если имя не задано, бот спрашивает, как обращаться к пользователю.
- Используется `preferred_name`, если оно сохранено.

### Структура обработчиков
- Старый монолитный обработчик разнесен по тематическим модулям:
  - `bot/handlers/single_message_parts/common.py`
  - `bot/handlers/single_message_parts/core.py`
  - `bot/handlers/single_message_parts/tasks.py`
  - `bot/handlers/single_message_parts/calendar.py`
  - `bot/handlers/single_message_parts/diary.py`
  - `bot/handlers/single_message_parts/health.py`
- `bot/handlers/single_message.py` оставлен как совместимый wrapper для подключения роутера.

## Что реализовано частично
- `Web App`: нижняя кнопка Telegram уже настроена, но полноценного frontend и backend API в репозитории пока нет.
- `RPG`: уровень, EXP и streak работают, но большая игровая система из `PLAN.md` еще не построена.
- `Здоровье`: уже есть вода, сон, тренировки, лекарства и `стресс / энергия`, но еще нет автоматических health-сценариев, рекомендаций и более глубокой аналитики.
- `Статистика`: есть периоды и агрегаты, но пока нет графиков и Web App-визуализации.

## Что логично делать дальше

2. Доработать `Статистику` под будущие графики и внешнюю визуализацию.
3. Вернуться к расширенному `Здоровью`: автоматические сценарии, рекомендации, health-челленджи.

## Короткий roadmap

### MVP
- жестко стабилизировать текущий Telegram UI и single-message lifecycle;
- держать закрытыми базовые сценарии по задачам, дневнику, здоровью, профилю и статистике;
- считать health-MVP закрытым по Telegram слою: вода, сон, тренировки, лекарства, состояние.

### v1
- добавить backend API для `Web App`;
- вынести в `Web App` календарь, графики и расширенную статистику;
- расширить health-модуль beyond MVP.

### v2
- расширить RPG-слой: инвентарь, экипировка, социальные механики, дуэли;
- добавить AI-блоки и внешние интеграции;
- перейти от MVP-бота к связке `бот + API + Web App`.

## Ограничения текущей версии
- Основной интерфейс рассчитан на личный чат с ботом.
- Reply keyboard Telegram нельзя "держать" в полностью пустом чате без нового исходящего сообщения от бота. После удаления истории восстановление идет через `/start`.
- Полного набора автотестов в репозитории пока нет.
- Есть `unittest`-контур на helper-, service- и ключевые Telegram UI flow, но это еще не end-to-end покрытие.
- `Web App` в README упоминается как точка интеграции, а не как готовый модуль.
- Визуальный слой интерфейса еще продолжает дорабатываться.

## Быстрый запуск

### 1. Подготовить переменные окружения
Создай `.env` на основе `.env.example` и заполни минимум:
- `BOT_TOKEN`
- `DATABASE_URL`
- `WEBAPP_URL`

### 2. Поднять PostgreSQL
```bash
docker compose up -d postgres
```

### 3. Применить миграции
```bash
alembic upgrade head
```

### 4. Запустить бота
```bash
python -m bot.main
```

Для Windows с локальным venv:
```powershell
.\.venv\Scripts\python.exe -m bot.main
```

## Полезные команды

```bash
python -m compileall bot backend alembic
alembic current
alembic history
.venv\Scripts\python.exe -m unittest discover -s tests -v
```
