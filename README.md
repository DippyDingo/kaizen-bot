# KAIZEN Bot

Telegram-бот в формате "одно обновляемое сообщение" + RPG-мотивация.

Документ отражает фактическое состояние проекта на **6 марта 2026**.

## Что уже реализовано

### 1) База и миграции
- Поднят async-слой БД на `SQLAlchemy` + `asyncpg`.
- Настроен `Alembic` и добавлены миграции:
  - `20260306_000001_init_mvp` (users/tasks/water/sleep/habits)
  - `20260306_000002_add_diary_entries`
  - `20260306_000003_expand_diary_media`
- Модели: `User`, `Task`, `WaterLog`, `SleepLog`, `Habit`, `DiaryEntry`.

### 2) Сервисы backend (MVP)
- `user_service`: создание/получение пользователя по `telegram_id`.
- `task_service`: создание задач, листинг по дате, переключение `done/undone`, удаление.
- `health_service`: лог воды, расчет воды за день, расчет сна за день, лог сна.
- `diary_service`: добавление записей, выборка/подсчет записей за день, получение записи по id.
- `rpg_service`: EXP-таблица, ап/даун уровней.

### 3) Интерфейс бота (single-message)
- Полностью кнопочное управление в одном обновляемом сообщении.
- Разделы: `Главная`, `Задачи`, `Календарь`, `Статистика`, `Здоровье`, `Дневник`.
- Навигация "внутри раздела + назад в меню".
- Нет ручной кнопки "обновить", интерфейс перерисовывается автоматически после действий.

### 4) Функционал по разделам
- **Задачи**
  - Добавление через FSM: `название -> приоритет -> дата (календарь)`.
  - Формат списка: `Задача | ✅ | ❌`.
  - Повторный тап на `✅` снимает выполнение.
- **Календарь**
  - Переключение месяцев/дней.
  - Выбор даты для просмотра задач.
  - Выбор даты при создании задачи.
- **Здоровье**
  - Быстрые кнопки воды `+150 / +250 / +500 мл`.
  - Запись воды на выбранную пользователем дату.
- **Дневник**
  - Поддержка: текст, голосовые, кружки, фото, видео.
  - Просмотр одной записи.
  - "Показать всё" за день (выгрузка всех записей в чат).
  - "Очистить" для удаления отправленных ботом сообщений выгрузки.
- **Главная/Статистика**
  - Виджеты HP/мана/стамина, уровень/EXP, streak, дневная сводка.

### 5) Рефакторинг структуры хендлеров
- Старый монолитный/разрозненный подход заменен на модульный:
  - `bot/handlers/single_message_parts/common.py`
  - `bot/handlers/single_message_parts/core.py`
  - `bot/handlers/single_message_parts/tasks.py`
  - `bot/handlers/single_message_parts/calendar.py`
  - `bot/handlers/single_message_parts/diary.py`
  - `bot/handlers/single_message_parts/health.py`
- `bot/handlers/single_message.py` оставлен как compatibility-wrapper.

## Быстрый запуск

### 1) Подготовка окружения
- Скопируй `.env.example` в `.env` и заполни значения.
- Убедись, что задан `BOT_TOKEN`.

Минимально важные переменные:
- `BOT_TOKEN`
- `DATABASE_URL` (по умолчанию настроен на локальный Postgres в Docker)
- `WEBAPP_URL` (можно оставить заглушку)

### 2) Поднять PostgreSQL
```bash
docker compose up -d postgres
```

### 3) Применить миграции
```bash
alembic upgrade head
```

### 4) Запустить бота
```bash
python -m bot.main
```

Для Windows (если используется локальный venv):
```powershell
.\.venv\Scripts\python.exe -m bot.main
```

## Полезные команды

```bash
# Проверка компиляции хендлеров
python -m compileall bot/handlers

# Текущая ревизия БД
alembic current

# Список миграций
alembic history
```
